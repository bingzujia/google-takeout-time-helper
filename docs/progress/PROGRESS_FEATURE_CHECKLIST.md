# gtoh migrate 进度条 — 功能验证清单

## 一、进度条显示

- [ ] **扫描阶段提示**：显示 `Scanning files...`
- [ ] **文件总数统计**：显示 `Found N files in M year folder(s)`
- [ ] **进度条格式**：`🔄 [+++++---------------] 25% (5/20)`
- [ ] **百分比正确**：`current * 100 / total` 计算准确
- [ ] **进度条长度**：20 字符，填充比例正确
- [ ] **回车覆盖**：使用 `\r` 覆盖当前行，不产生多行输出
- [ ] **完成后换行**：处理完成后输出一个换行符

## 二、更新频率

- [ ] **< 1000 文件**：每个文件更新一次进度条
- [ ] **>= 1000 文件**：每 10 个文件更新一次
- [ ] **最后一个文件**：无论总数多少，最后一条必须更新到 100%
- [ ] **不闪烁**：终端无明显闪烁现象

## 三、边界场景

- [ ] **0 文件**：显示 `No media files found in year folders.`，不显示进度条
- [ ] **1 文件**：进度条直接显示 100%
- [ ] **100 文件**：每个文件更新，进度条平滑
- [ ] **1000 文件**：每 10 个更新一次
- [ ] **10000 文件**：每 10 个更新一次，不卡顿

## 四、终端兼容性

- [ ] **标准终端**：进度条正常显示
- [ ] **管道输出**：`\r` 字符不影响管道数据
- [ ] **重定向到文件**：进度条字符写入文件但不影响可读性
- [ ] **CI 环境**：不报错，正常完成

## 五、与日志协调

- [ ] **log 文件不受影响**：进度条不写入 gtoh.log
- [ ] **SKIP 记录**：跳过的文件记录到 log，不在终端打印
- [ ] **FAIL 记录**：失败的文件记录到 log，不在终端打印
- [ ] **最终统计**：终端输出完整统计，包含所有跳过/失败分类

## 六、配置开关

- [ ] **ShowProgress 默认 true**：默认显示进度条
- [ ] **ShowProgress = false**：不显示进度条，仅显示扫描和统计信息
- [ ] **命令行无额外参数**：进度条默认开启

## 七、统计一致性

- [ ] **scanned 与进度条最终值一致**
- [ ] **processed + skipped + failed = scanned**
- [ ] **最终统计数字准确**
