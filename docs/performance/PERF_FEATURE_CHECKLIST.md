# gtoh 性能优化 — 功能验证清单

## 方案 1：并发处理

- [ ] **并发数配置**：默认 `min(runtime.NumCPU(), 8)`
- [ ] **并发数可配置**：可通过参数调整
- [ ] **stats 线程安全**：Scanned/Processed 等计数器正确累加
- [ ] **logger 线程安全**：log 文件写入不出现乱码或丢失
- [ ] **进度条线程安全**：进度条更新不闪烁、不重叠
- [ ] **错误目录线程安全**：error 目录并发创建不冲突
- [ ] **metadata 目录线程安全**：metadata JSON 并发写入不冲突
- [ ] **完成等待**：所有文件处理完成后才输出统计
- [ ] **性能验证**：1000 文件处理时间 < 串行时间的 1/4（4 核 CPU）

## 方案 2：合并 exiftool 读取

- [ ] **时间戳提取**：合并调用后 DateTimeOriginal 正确解析
- [ ] **GPS 提取**：合并调用后 GPSLatitude/GPSLongitude 正确解析
- [ ] **优先级不变**：EXIF > Filename > JSON 优先级逻辑不变
- [ ] **回退机制**：合并调用失败时回退到单独调用
- [ ] **性能验证**：exiftool 进程启动次数减少 50%

## 方案 3：流式复制 + SHA-256

- [ ] **复制正确**：文件内容与源文件一致
- [ ] **SHA-256 正确**：与单独计算的结果一致
- [ ] **大文件处理**：500MB 视频正确处理，内存不爆炸
- [ ] **冲突检测**：目标文件已存在时正确返回 exists=true
- [ ] **性能验证**：大文件 I/O 时间减半

## 方案 4：合并 file 命令

- [ ] **类型检测**：MIME 类型正确识别
- [ ] **扩展名映射**：mimeToExt 映射正确
- [ ] **支持检测**：WMV 正确识别为不支持
- [ ] **性能验证**：file 进程启动次数减少 50%

## 方案 5：跳过不必要的 exiftool 读取

- [ ] **文件名优先**：文件名可解析时不调用 exiftool
- [ ] **EXIF 回退**：文件名无法解析时正确调用 exiftool
- [ ] **优先级不变**：EXIF > Filename > JSON 优先级逻辑不变
- [ ] **metadata 记录**：timestamp 来源正确记录
- [ ] **性能验证**：文件名可解析的文件节省 exiftool 调用

## 组合验证

- [ ] **全部方案实施后**：10000 文件处理时间 < 原始时间的 1/5
- [ ] **功能不变**：输出文件、metadata、log 与优化前一致
- [ ] **错误处理不变**：失败文件正确移到 error 目录
- [ ] **进度条正常**：并发下进度条平滑更新
